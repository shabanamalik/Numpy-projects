🟢 Beginner Level
Q1
Beginner
What is the difference between a Python list [1,2,3] and a NumPy array np.array([1,2,3])? Give at least 3 differences.
+

Key Differences:

1. Homogeneous vs Heterogeneous: NumPy arrays store elements of ONE type (int64, float32). Python lists can mix types — [1, "hello", 3.14].

2. Speed: NumPy operations run in compiled C code — 50–100× faster because no type-checking per element.

3. Memory: NumPy stores data in one contiguous block. Lists store pointers to scattered objects — uses 5–20× more RAM.

4. Vectorized Math: arr * 2 doubles every element. With a list, you need a loop or list comprehension.

5. Multi-dimensional: NumPy natively supports 2D, 3D, nD arrays with shape/axis operations. Lists need nesting.

proof of speed difference
# Python list vs NumPy – timing comparison
import numpy as np, time

lst = list(range(1_000_000))
arr = np.arange(1_000_000)

t1 = time.time()
result = [x * 2 for x in lst]
print(f"List: {time.time()-t1:.4f}s")    # ~0.08s

t2 = time.time()
result = arr * 2
print(f"NumPy: {time.time()-t2:.4f}s")   # ~0.001s → 80x faster!

Q7

What is the difference between np.copy() and a slice? Write code to prove it.
+
view vs copy – critical concept
original = np.array([1, 2, 3, 4, 5])

# Slicing creates a VIEW (shared memory!)
view = original[1:4]
view[0] = 999
print(original)   # [1 999 3 4 5]  ← ORIGINAL CHANGED!

original = np.array([1, 2, 3, 4, 5])

# copy() creates independent data
safe_copy = original[1:4].copy()
safe_copy[0] = 999
print(original)   # [1 2 3 4 5]  ← UNCHANGED

print(np.shares_memory(original, safe_copy))  # False

AI Warning: In data preprocessing pipelines, accidentally modifying the original dataset through views is a common and hard-to-debug bug. Always use .copy() when you intend to modify a slice.

You have a matrix of raw sensor readings (5×4). Normalize each column to have mean=0 and std=1 using broadcasting. This is called Z-score normalization.

z-score normalization – used in EVERY ML pipeline
np.random.seed(1)
data = np.random.randint(20, 100, size=(5, 4)).astype(float)
print("Raw data:\n", data)

mean = np.mean(data, axis=0)   # shape (4,) → column means
std  = np.std(data, axis=0)    # shape (4,) → column stds

normalized = (data - mean) / std  # broadcasting: (5,4)-(4,) works!
print("Normalized:\n", normalized)

print("Check stds ≈ 1: ", np.round(np.std(normalized, axis=0), 10))

Q15
Intermediate
Predict whether these broadcasting operations will work or fail. Then verify: (a) shapes (3,4) + (4,), (b) shapes (3,1) + (1,4), (c) shapes (3,4) + (3,), (d) shapes (4,) + (3,4)
+
broadcasting rules
# Rule: align shapes from RIGHT. Dimensions must be equal OR one of them is 1.

# a) (3,4) + (4,)  → (4,) treated as (1,4) → broadcasts to (3,4) ✅ WORKS
a = np.ones((3, 4)) + np.ones(4)
print(a.shape)   # (3, 4) ✅

# b) (3,1) + (1,4) → broadcasts to (3,4) ✅ WORKS
b = np.ones((3,1)) + np.ones((1,4))
print(b.shape)   # (3, 4) ✅

# c) (3,4) + (3,)  → align right: 4 vs 3 → NOT equal, neither is 1 ❌ FAILS
# np.ones((3,4)) + np.ones(3)  → ValueError!

# d) (4,) + (3,4) → (4,) becomes (1,4), broadcasts to (3,4) ✅ WORKS
d = np.ones(4) + np.ones((3,4))
print(d.shape)   # (3, 4) ✅
Q16
Advanced
Implement the sigmoid activation function and ReLU using only NumPy. Apply them to a batch of 10 random values.
+
neural network activations from scratch
np.random.seed(42)
z = np.random.randn(10)  # raw neuron outputs
print("Input z:", np.round(z, 3))

# Sigmoid: squashes to (0, 1)
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

# ReLU: max(0, z)
def relu(z):
    return np.maximum(0, z)

# Softmax: converts to probability distribution
def softmax(z):
    e_z = np.exp(z - np.max(z))  # subtract max for numerical stability!
    return e_z / np.sum(e_z)

print("Sigmoid:", np.round(sigmoid(z), 3))  # all between 0 and 1
print("ReLU:   ", np.round(relu(z), 3))     # negatives become 0
print("Softmax:", np.round(softmax(z), 3))   # sums to 1.0
print("Softmax sum:", softmax(z).sum())       # 1.0

This is what happens INSIDE every neural network, every forward pass! TensorFlow and PyTorch are just wrappers around exactly this.

data = np.array([[100, 0.5, 3000],
                  [200, 1.2, 1500],
                  [50,  0.1, 5000],

Q19
Advanced
Implement a single neural network forward pass using only np.dot. Input layer: 3 features, hidden layer: 4 neurons, output layer: 2 classes.
+
neural network forward pass – pure numpy
np.random.seed(0)

# Architecture: 3 → 4 → 2

# Input: batch of 5 samples, each with 3 features
X = np.random.randn(5, 3)         # shape (5, 3)

# Layer 1 weights and bias
W1 = np.random.randn(3, 4) * 0.1   # shape (3, 4)
b1 = np.zeros((1, 4))               # shape (1, 4)

# Layer 2 weights and bias
W2 = np.random.randn(4, 2) * 0.1   # shape (4, 2)
b2 = np.zeros((1, 2))               # shape (1, 2)

# Forward pass
Z1 = X @ W1 + b1                    # (5,3)@(3,4)+(1,4) = (5,4)
A1 = np.maximum(0, Z1)             # ReLU activation

Z2 = A1 @ W2 + b2                   # (5,4)@(4,2)+(1,2) = (5,2)
A2 = 1 / (1 + np.exp(-Z2))         # Sigmoid – output probabilities

print("Output probabilities:\n", np.round(A2, 4))
print("Shape:", A2.shape)              # (5, 2) → 5 samples, 2 class probs

# Predicted class
predictions = np.argmax(A2, axis=1)
print("Predictions:", predictions)

This IS what TensorFlow/PyTorch do under the hood. Understanding this in NumPy makes you a better AI engineer.

Implement matrix multiplication from scratch using only NumPy array indexing (no np.dot). Verify against np.dot.
+
matrix multiply from scratch
def matmul_scratch(A, B):
    m, k = A.shape
    k2, n = B.shape
    assert k == k2, "Incompatible shapes!"
    C = np.zeros((m, n))
    for i in range(m):
        for j in range(n):
            C[i, j] = np.sum(A[i, :] * B[:, j])  # dot of row i, col j
    return C

A = np.random.randint(1, 10, (3, 4))
B = np.random.randint(1, 10, (4, 2))

result = matmul_scratch(A, B)
print("Matches np.dot:", np.allclose(result, np.dot(A, B)))  # True

Q22

What is vectorization? Rewrite this slow Python loop using vectorized NumPy operations:
result = [x**2 + 2*x + 1 for x in range(1000000)]
+
vectorization – the NumPy way
import time

# Slow Python loop
t1 = time.time()
result_loop = [x**2 + 2*x + 1 for x in range(1_000_000)]
print(f"Loop: {time.time()-t1:.3f}s")       # ~0.12s

# Fast vectorized NumPy
t2 = time.time()
x = np.arange(1_000_000)
result_np = x**2 + 2*x + 1               # applies to ALL elements at once
print(f"NumPy: {time.time()-t2:.3f}s")     # ~0.003s → 40x faster!

# Verify same results
print(np.allclose(result_loop, result_np))  # True

Golden Rule: If you find yourself writing a Python for loop over NumPy arrays, STOP. There is almost always a vectorized way. Loops + NumPy = slow. Pure NumPy operations = fast.

Q23
Expert
Explain and demonstrate np.einsum. Use it to compute: (a) matrix multiply, (b) batch matrix multiply, (c) trace of a matrix.
+
einsum – used heavily in transformers/attention
A = np.random.rand(3, 4)
B = np.random.rand(4, 5)

# a) Matrix multiply: (3,4) @ (4,5) = (3,5)
C = np.einsum('ij,jk->ik', A, B)    # i,k are free; j is summed
print(np.allclose(C, A @ B))          # True

# b) Batch matrix multiply: (batch, n, m) @ (batch, m, p)
batch_A = np.random.rand(10, 3, 4)
batch_B = np.random.rand(10, 4, 5)
batch_C = np.einsum('bij,bjk->bik', batch_A, batch_B)
print(batch_C.shape)                  # (10, 3, 5)

# c) Trace (sum of diagonal elements)
square = np.random.rand(4, 4)
trace = np.einsum('ii->', square)    # same i repeated = diagonal
print(trace == np.trace(square))      # True

einsum powers the attention mechanism in Transformers (GPT, BERT). If you want to build AI models from scratch, learn einsum deeply.

Q24
Expert
Implement gradient descent for linear regression using only NumPy. Fit a line to 50 noisy data points.
+
linear regression with gradient descent – pure numpy
np.random.seed(42)
n = 50

# True line: y = 3x + 7 + noise
X = np.random.rand(n, 1) * 10
y = 3 * X + 7 + np.random.randn(n, 1) * 2

# Initialize weights
w, b = np.zeros((1, 1)), np.zeros((1, 1))
lr = 0.01

for epoch in range(1000):
    # Forward pass: y_hat = X*w + b
    y_hat = X @ w + b

    # Loss (MSE)
    loss = np.mean((y_hat - y) ** 2)

    # Gradients (calculus)
    dw = (2/n) * X.T @ (y_hat - y)
    db = (2/n) * np.sum(y_hat - y)

    # Update weights
    w -= lr * dw
    b -= lr * db

    if epoch % 200 == 0:
        print(f"Epoch {epoch}: loss={loss:.4f}, w={w[0,0]:.2f}, b={b[0,0]:.2f}")

print(f"\nFinal: w≈{w[0,0]:.2f} (true=3.0), b≈{b[0,0]:.2f} (true=7.0)")

Critical

np.linalg – Linear Algebra

Inverse, determinant, eigenvalues, SVD, matrix rank, solve linear equations. The mathematical foundation of PCA, neural nets, and recommender systems.

Einstein summation notation — powers Transformer attention mechanisms. Used in every modern LLM implementation including GPT, BERT, and ViT.

Row-major vs column-major memory storage. Affects performance dramatically in large matrix operations. np.ascontiguousarray() is your friend.

Important

np.pad

Padding arrays with zeros, reflections, or constants. Used in every CNN for image border handling during convolution operations.

Important

Arrays with mixed named dtypes — like a lightweight database table. Used in efficient data loading and feature engineering.

Useful

np.unique, np.bincount

Count class frequencies, find unique labels. Essential for class imbalance detection in classification problems.

Important

np.clip

Clamping values to a range. Used in image preprocessing (0–255), gradient clipping in training, and bounding box coordinates in object detection.

Important

np.apply_along_axis

Apply a function to each row or column efficiently. A clean alternative to loops for custom per-sample operations.

Adding dimensions to match expected tensor shapes. You'll use this EVERY DAY when feeding data into neural networks: x[np.newaxis, :].

Useful

Create coordinate grids for 2D plots, decision boundary visualization, and image coordinate systems in computer vision.

Persist arrays to disk as .npy files — fast binary format. Also np.savez for multiple arrays. Essential for caching preprocessed data.

W2 = np.random.randn(4, 1) * 0.5
b2 = np.zeros((1, 1))

sigmoid = lambda z: 1 / (1 + np.exp(-z))
lr = 0.5

for epoch in range(10000):
    # Forward
    A1 = sigmoid(X @ W1 + b1)
    A2 = sigmoid(A1 @ W2 + b2)
    loss = np.mean((A2 - y)**2)

    # Backward (chain rule)
    dA2 = 2*(A2-y)/len(y) * A2*(1-A2)
    dW2 = A1.T @ dA2
    dA1 = dA2 @ W2.T * A1*(1-A1)
    dW1 = X.T @ dA1

    W2 -= lr*dW2; b2 -= lr*dA2.sum(axis=0, keepdims=True)
    W1 -= lr*dW1; b1 -= lr*dA1.sum(axis=0, keepdims=True)

    if epoch % 2000 == 0:
        print(f"Epoch {epoch}: loss={loss:.4f}")

print("Predictions:", np.round(A2, 2))
# Should converge to [[0], [1], [1], [0]]

def convolve2d(image, kernel):
    kh, kw = kernel.shape
    pad_h, pad_w = kh // 2, kw // 2
    padded = np.pad(image, ((pad_h, pad_h), (pad_w, pad_w)), mode='constant')
    H, W = image.shape
    output = np.zeros_like(image, dtype=float)
    for i in range(H):
        for j in range(W):
            region = padded[i:i+kh, j:j+kw]
            output[i, j] = np.sum(region * kernel)  # element-wise + sum
    return np.clip(output, 0, 255)

    # X shape (120,2), centroids (3,2) → distances (120,3)

    new_centroids = np.array([X[labels==i].mean(axis=0) for i in range(k)])

    # Convergence check
    if np.allclose(centroids, new_centroids): 
        print(f"Converged at iteration {iteration}")
        break
    centroids = new_centroids

print("Final centroids:\n", centroids)
print("Cluster sizes:", [np.sum(labels==i) for i in range(k)])

    [5, 4, 0, 0, 1, 0, 0, 3],
    [4, 0, 4, 1, 0, 0, 0, 0],
    [0, 0, 0, 5, 4, 5, 0, 0],

    [1, 0, 5, 0, 0, 0, 5, 4],
], dtype=float)

# User similarity using cosine similarity
def cosine_sim(A):
    norms = np.linalg.norm(A, axis=1, keepdims=True)
    norms[norms == 0] = 1
    A_norm = A / norms
    return A_norm @ A_norm.T

sim = cosine_sim(R)       # 5×5 user similarity matrix
print("User similarity:\n", np.round(sim, 2))

# Recommend movies for user 0
user = 0
not_seen = R[user] == 0   # movies user hasn't watched

# Weighted average ratings from similar users
user_sims = sim[user]
predicted = user_sims @ R / (user_sims.sum() + 1e-8)
predicted[~not_seen] = 0   # only consider unseen

top_recs = np.argsort(predicted)[::-1][:3]
print(f"Top 3 movie recommendations for user {user}: {top_recs}")
